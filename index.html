HEXO+Github搭建属于自己的博客

经过各种找资料，踩过何种坑，终于搭建好了hexo，域名目前用得时github的，我的hexo是3.2.2版本，hexo不同的版本，很多配置都不一样。

为了做一个合格的程序员，这几天我也尝试搭建自己的博客。很自然的选择github来搭建，因为免费嘛，对大多数人来说应该是入门博客最好的选择了。
搭博客自然有样式选择，在网上搜索后，发现很多人都是选择Hexo框架来搭建的。
网上参考潘柏信同学的博客教程（原文地址http://baixin.io/2015/08/HEXO%E6%90%AD%E5%BB%BA%E4%B8%AA%E4%BA%BA%E5%8D%9A%E5%AE%A2/）开始搭建自己的博客。安装好相应的环境（在安装nodejs的时候按照博主的方法进行安装安装失败了，后来下载安装包直接安装了，这里就不细说了），输入命令hexo server后，浏览器访问http://localhost:4000后能看到博客页面，有些小小激动。接下来就是要将hexo主题内容提交到自己的github上面，这一步就被坑惨了，出现各种奇奇怪怪的问题，搞了好久之后才搞好。
首先在cmd中执行hexo d命令，将hexo主题发布到github是报错，如下图：

可以看到，报了个FATAL spawn Enoent的错误，提交报错，所以猜测是hexo-deployer-git插件的问题，所以尝试到到该github上面是看看有没有人遇到这样的问题，果然有这个issue，回答说的很清楚，我安装了github for windows但是也安装了git，但是在cmd窗口中执行git命令确提示没有该命令，然后我发现有个git cmd的工具，于是尝试在这个窗口里面去执行deploy命令，执行后还是报错了，单这次报的错和之前完全不一样了，错误如下图：

报的是could not read username for ‘https://github.com':no error的错误，网上直接搜索这个错误，讨论有很多啊，有的说将https的方式换成ssh的方式（试过，报的另外一个错），后来看到一个回答说将url的结构改一下，抱着试一试的态度尝试了一下，居然成功了！阿西吧，不容易啊。然后这里有个小小的思考，hexo git deployer说明的repository url说的应该就是github上直接copy下来的repository url啊，具体原因有待考究。
这样，hexo+github搭建的博客就算起来啦，有些简陋，但也算是有个小屋啦，后面再慢慢完善吧。
